# K-Food Export Passport AI

## Hero 섹션 & 로고 비주얼 고도화 프롬프트 (이미지 중심)

**목적**  
현재 Landing Page는 구조와 문구는 충분히 완성되었으나,  
비주얼 요소(이미지, 로고 아이콘)의 임팩트가 약해  
2024–2025년 기준 B2B AI SaaS 서비스 대비 다소 정적인 인상을 준다.

본 문서는 **기능/문구/레이아웃 변경 없이**,  
이미지와 로고 비주얼만으로 체감 품질을 끌어올리는 것을 목표로 한다.

---

## 1. 전제 조건 (절대 변경 금지)

- Hero 섹션의 텍스트 문구 유지
- CTA 버튼 문구 및 위치 유지
- 전체 레이아웃 구조 유지
- 기존 다크모드 디자인 퀄리티 유지
- 새로운 기능 추가 금지

→ **비주얼 개선만 허용**

---

## 2. Hero 섹션 비주얼(이미지) 개선 지시

### 2-1. 개선 목표

- Hero를 “설명 카드”가 아닌  
  **AI가 실제로 분석을 수행하고 있다는 첫 인상 공간**으로 전환
- 사용자가 CTA 클릭 시  
  “이런 결과가 나오겠구나”를 직관적으로 상상하게 만들 것

---

### 2-2. 이미지 방향성

다음 중 하나 또는 조합을 활용한다.

#### A. 서비스 결과 암시형 이미지

- 식품 라벨 이미지 위에
  - 체크 아이콘
  - 경고 아이콘
  - 하이라이트 박스
    가 UI 오버레이 형태로 얹힌 이미지
- 실제 서비스 스크린샷처럼 보이되, 디테일은 단순화

#### B. AI 분석 흐름 추상 비주얼

- Image → OCR → Regulation → Result 흐름을
  선, 점, 글로우, 미니 다이어그램 형태로 표현
- 기술적이되 설명적이지 않게

#### C. 보조 비주얼 배치

- Hero 카드 내부 우측 또는 하단에
  작은 미리보기 이미지 형태로 배치
- 텍스트와 CTA를 방해하지 않을 것

---

### 2-3. 배치 원칙

- Hero의 주인공은 **텍스트 + CTA**
- 이미지는 “보조 증거” 역할
- 모바일에서는 이미지가 축소되거나 제거되어도 자연스러울 것

---

## 3. 로고 비주얼(아이콘) 개선 지시

### 3-1. 현재 상태 인식

- 로고 **텍스트(K-Food Export Passport)**는 만족스러운 상태
- 아이콘(심볼)은 다소 일반적이며
  서비스의 전문성과 AI SaaS 이미지를 충분히 전달하지 못함

---

### 3-2. 로고 아이콘 개선 목표

- 텍스트는 유지
- 아이콘만 교체 또는 리디자인
- 다음 키워드를 시각적으로 암시할 것:
  - Export / Passport
  - Verification / Check
  - AI / Automation
  - Trust / Authority (B2B)

---

### 3-3. 아이콘 디자인 방향

- 단색 또는 2톤 컬러
- 작은 사이즈(헤더)에서도 식별 가능
- 추상적이되 의미 없는 도형은 피할 것

**허용 예시**

- 패스포트 + 체크
- 문서 + 스캔 라인
- 라벨 태그 + 인증 마크
- 심플한 심볼형 AI 노드

**금지**

- 캐릭터형 아이콘
- 음식 일러스트
- 과도하게 복잡한 SVG
- 스타트업 느낌의 귀여운 로고

---

## 4. 다크 / 라이트 모드 대응 원칙

- 기본 테마: Light Mode
- 보조 테마: Dark Mode
- 로고 아이콘과 Hero 이미지는
  두 테마 모두에서 동일한 정체성을 유지해야 함
- 다크모드에서는
  - subtle glow
  - soft border
    허용
- 라이트모드에서는
  - 그림자 또는 명확한 대비로 존재감 확보

---

## 5. 금지 사항 요약

- 스톡 인물 사진 사용 금지
- Hero 전체를 이미지로 덮는 방식 금지
- 과장된 AI 일러스트 금지
- 기능을 오해하게 만드는 비주얼 금지

---

## 6. 결과물 요청

- Hero에 추가된 이미지/비주얼 설명
- 로고 아이콘이 기존 대비 어떻게 더 전문적으로 보이는지 설명
- 이 변경이 첫 방문 사용자의 CTA 클릭에
  어떤 도움을 주는지 한 문장 요약
