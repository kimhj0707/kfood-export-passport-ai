# UI / UX 디자인 고도화 가이드

**K-Food Export Passport AI**

**작성일:** 2026-01-16  
**문서 성격:** 디자인 기준 고정 문서 (전체 페이지 공통 적용)

---

## 1. 현재 문제 인식 (중요)

현재 서비스는 기능적으로는 MVP 기준을 충족했으나,  
UI/UX 측면에서는 다음과 같은 문제가 존재한다.

### 1-1. 전체적인 인상

- 오래된 기본 웹사이트처럼 보임
- 최신 SaaS 서비스의 시각적 밀도 부족
- 기능 설명 위주의 레이아웃으로 감성적 신뢰 부족

### 1-2. 랜딩 페이지 문제

- 첫 화면에서 “요즘 서비스” 느낌이 약함
- 배경, 카드, CTA가 평면적
- 기술 기반 서비스라는 인상이 즉시 전달되지 않음

### 1-3. 다크 모드 문제 (심각)

- 페이지별로 다크 모드 적용 수준이 제각각
- 일부 텍스트 가독성 저하
- 배경은 다크인데 카드/보더는 라이트 기준 색상 사용
- 상태 UI(모달, 토스트, 로딩)는 다크모드 미대응

---

## 2. 디자인 고도화 목표 (방향 고정)

### 핵심 목표

> “2024~2025년에 출시된 B2B AI SaaS 서비스처럼 보이게 만든다.”

### 금지 방향

- 전통적인 기업 홈페이지 스타일 ❌
- Bootstrap 느낌 ❌
- 표 중심 레이아웃 ❌
- 과도한 설명 텍스트 ❌

---

## 3. 전체 공통 디자인 원칙 (랜딩 포함)

### 3-1. 디자인 키워드

- Modern B2B SaaS
- AI / Tech / Trust
- Dark Mode First
- Minimal & High Contrast
- Depth 있는 카드 레이아웃

---

### 3-2. 컬러 시스템 (기준)

**Background**

- 기본 배경: `slate-950 ~ slate-900`
- Gradient 허용 (단, subtle)

**Primary**

- Indigo / Blue 계열
- CTA는 항상 대비가 확실해야 함

**Text**

- 기본 텍스트: white / white-90%
- 보조 텍스트: white-60%
- 설명 텍스트 남용 금지

---

### 3-3. 카드 / 컨테이너 규칙

모든 카드형 UI는 아래 기준을 따른다.

- `rounded-xl` 이상
- border는 `border-white/10`
- 배경은 `bg-white/5`
- `backdrop-blur` 허용
- 그림자는 `shadow-lg` 이상

> ❌ 회색 박스, ❌ 얇은 실선 금지

---

## 4. 랜딩 페이지 디자인 방향

### 4-1. Hero Section

**필수 요소**

- 강한 첫 문장 (짧고 명확)
- 부제는 1~2줄 이내
- CTA 1개만 강조

**디자인**

- Gradient 배경
- 카드 or 글래스모피즘 요소
- 중앙 정렬 또는 좌우 대비 구조

---

### 4-2. 문제 정의 섹션

- “왜 이 서비스가 필요한가”를 시각적으로 표현
- 텍스트 나열 ❌
- 카드 2~3개로 요약

---

### 4-3. 프로세스 설명

- 기존 3단계 유지
- 리스트 ❌
- 시각적 단계 표현 ⭕
- 아이콘 또는 강조 포인트 사용

---

## 5. 랜딩 외 전체 페이지 공통 개선 대상

### 5-1. Analyze Page

- 로딩 모달: 다크모드 전용 스타일 필요
- 진행 상태 UI에 depth 부족
- 상태 완료 아이콘 색상 통일 필요

---

### 5-2. Report Page

- 정보가 많아 “문서”처럼 보임
- 섹션 간 시각적 구분 부족
- 카드 단위 재구성 필요
- 전문가 뷰 / 일반 뷰 시각적 차별 강화

---

### 5-3. History Page

- 리스트형 UI가 너무 단순
- SaaS 대시보드 느낌 부족
- hover / focus 상태 미흡

---

## 6. 다크 모드 정비 기준 (중요)

### 6-1. 원칙

- Light / Dark 분기 ❌
- Dark Mode First ⭕
- 모든 컴포넌트는 다크 기준으로 설계

---

### 6-2. 반드시 점검할 UI

- 모달
- 토스트
- 드롭다운
- 버튼 hover / active
- 테이블 헤더
- 입력 필드 focus 상태

---

### 6-3. 금지 사례

- 흰 배경 위 흰 텍스트 ❌
- 라이트모드 색상 그대로 사용 ❌
- opacity 없는 회색 텍스트 ❌

---

## 7. 구현 방식 가이드 (Tailwind 기준)

- 유틸리티 클래스 중복 최소화
- 공통 스타일은 컴포넌트화
- 페이지별 개별 색상 정의 금지
- 색상은 반드시 디자인 기준에 맞출 것

---

## 8. 작업 우선순위

### Priority 1

- 랜딩 페이지 Hero 섹션
- 전체 배경 / 카드 스타일 통일

### Priority 2

- Report Page 카드 구조 재정비
- 다크 모드 컬러 정합성 수정

### Priority 3

- Analyze / History 페이지 시각 밀도 개선
- SaaS 대시보드 느낌 강화
