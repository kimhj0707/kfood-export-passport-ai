# K-Food Export Passport AI

## 2차 고도화 작업 지시서 (신뢰·설명·확장 증명)

**작성일**: 2026-01-16  
**단계 정의**: 기능 추가 ❌ / 구조 변경 ❌ / 신뢰·설명·확장성 고도화 ⭕  
**대상**: Gemini (조사·정리·구조화 전담)

---

## 1. 2차 고도화의 목적

본 프로젝트는 이미 서비스 MVP 1차 완성 단계에 도달했다.  
2차 고도화의 목적은 **기능을 늘리는 것이 아니라**,  
아래 3가지를 구조적으로 증명하는 데 있다.

1. 규정 근거의 **신뢰성**
2. AI 분석 결과의 **설명 가능성**
3. 산업·국가 확장의 **구조적 가능성**

---

## 2. 현재 상태 요약 (기준선)

- 규정 경고 시 `details` 객체를 통해
  - regulation
  - article
  - reason
    을 제공 중
- 국가별 규정 데이터는 구조는 검증되었으나,
  **출처·신뢰도·변동 가능성 표현이 부족**

---

## 3. 2차 고도화 핵심 과제

### 과제 A. 규정 근거의 “출처 레벨” 고도화

#### 목표

- “이 규정은 어디서 온 것인가?”에 한 문장으로 답변 가능

#### 작업 지시

각 국가별 규정(`details`)에 다음 필드를 반드시 추가할 것.

```json
"details": {
  "regulation": "공식 법령 또는 가이드명",
  "article": "조항 번호 또는 섹션명",
  "reason": "해당 규정의 핵심 취지 요약 (단정 금지)",
  "authority": "관할 기관 (예: FDA, CAA, MOH 등)",
  "source_type": "official_law | official_guidance | government_notice",
  "as_of": "기준 연도 또는 문서 발행/개정 시점",
  "confidence_level": "high | medium",
  "note": "해석/개정/변동 가능성 설명 (있는 경우)"
}
단정 표현 금지

숫자(알레르기 개수 등)는 변동 가능할 경우 반드시 note에 명시

과제 B. “설명 가능한 AI” 관점 보강
목표
AI가 왜 이런 경고를 냈는지 구조적으로 설명

작업 지시
아래 항목을 개념적으로 정리하고, 문서용 설명 문장을 생성할 것.

규정 경고는 판단이 아니라 규칙 매칭 결과임

OCR 텍스트 중 어떤 정보가 규칙과 연결되었는지 설명 가능해야 함

AI 오판 가능성을 전제로 한 보조 도구 포지션 명확화

출력 요청 (문장)
개발일지/발표/README에 공통으로 쓸 수 있는
“설명 가능성(Explainability) 요약 문장” 3종

과제 C. AI 책임 한계(disclaimer)의 구조화
목표
법적·규제 책임 회피를 UI 문구가 아닌 시스템 설계로 증명

작업 지시
아래와 같은 disclaimer 구조를 제안하고,
국가 공통으로 사용할 수 있는 표현을 정리할 것.

json
코드 복사
"disclaimer": {
  "type": "regulatory_support",
  "text": "This result is for preliminary compliance review support and does not constitute legal advice."
}
표현은 강하지 않게

“지원(support)” / “보조(preliminary)” 중심

과제 D. 확장 가능성의 구조적 증명
목표
“K-푸드 → K-뷰티 → 기타 산업” 확장이 말이 아니라 구조임을 증명

작업 지시
현재 규정 JSON 스키마가

식품

화장품

반려동물 사료
등 성분/표기 기반 산업에 공통 적용 가능함을 설명

실제 구현 없이,
확장 예시 JSON 1~2개만 생성

json
코드 복사
{
  "industry": "cosmetics",
  "risk": "Prohibited ingredient detected",
  "regulation": "EU Cosmetics Regulation",
  "article": "Annex II",
  "reason": "Listed substances must not be present in cosmetic products"
}

5. 2차 고도화 종료 조건
```
